| ## 背景<br>> 说明项目要解决的是什么人，在什么场景下，碰到什么问题，要解决什么问题 | 1.  **用户备考场景痛点**<br>    <br>    1.  雅思已全面切换为机考模式，且考试规则与交互细节持续迭代（如写作字数统计、听力答题界面、口语考试流程更新），学生缺乏高度还原最新机考环境的模拟平台，无法精准适配真实考试的操作流程、时间节奏与交互体验。<br>        <br>2.  **业务开发效率痛点**<br>    <br>    1.  唯寻杯每年固定开展 R1-R4 共 4 轮考试，每轮上线均需投入技术资源进行定制开发与测试，重复劳动成本高、迭代周期长。<br>        <br>    2.  业务方无法自主调整考试配置，所有规则变更都依赖技术排期，影响考试运营的灵活性与响应速度。 |
| --- | --- |
| ## 目标<br>> 要达成的目标是什么，要尽量拿出具体的依据和量化指标，说明是一个真实问题 | 1.  **场景覆盖目标**<br>    <br>    1.  同步雅思官方最新机考规则与交互细节，提供与真实考试 1:1 还原的线上模拟环境，覆盖听、说、读、写全流程，适配官方迭代的操作逻辑、界面布局与评分反馈机制。<br>        <br>    2.  支撑雅思机考、牛剑笔试模考、语培类模考等多类型考试场景，确保模拟环境与官方考试体验高度一致，解决学生因不熟悉机考流程导致的发挥失常问题。<br>        <br>2.  **平台化效率目标**<br>    <br>    1.  完成雅思机考平台的配置化改造，提供可视化运营配置页面，支持业务方自主完成雅思考试时间、报名规则、答题流程、防作弊策略、界面样式等配置，并可在线验证配置生效，无需依赖技术开发。<br>        <br>3.  **系统复用目标**<br>    <br>    1.  实现一套底层架构支撑多类型考试（唯寻杯、step、牛剑模考、雅思等），避免重复开发，提升系统扩展性与维护性，为后续新增考试类型提供快速接入能力。 |

# 业务流程

> 以时序图、流程图或者文字形式描述业务实际流程；

```mermaid
stateDiagram-v2
    [*] --> Draft : 创建考试
    Draft --> Published : 发布考试(配置校验通过)
    Published --> NotStarted : 考试未开始(考试筹备中)
    NotStarted --> InProgress : 考试进行中(考生进入)
    InProgress --> Submitted : 考生主动交卷(完成/超时)
    InProgress --> AutoSubmitted : 系统自动交卷(超时/中断)
    Submitted --> Marking : 进入阅卷(后台管理)
    AutoSubmitted --> Marking : 进入阅卷(后台管理)
    Marking --> Scored : 阅卷完成(成绩录入/生成)
    Scored --> Completed : 考试结束(报告生成/展示)
    Completed --> Archived : 归档(手动/定时)

    Draft --> Draft : 编辑考试
    Published --> Draft : 撤销发布
    InProgress --> Paused : 暂停考试(配置/紧急)
    Paused --> InProgress : 恢复考试

    state Draft {
        [*] --> BaseConfig
        BaseConfig --> IELTSConfig : 选择考试类型=IELTS
        IELTSConfig --> OrderConfig : 配置组合卷顺序
        OrderConfig --> RuleValidation : 校验官方题量/时长/Part/提交规则
        RuleValidation --> [*]
    }

    state InProgress {
        [*] --> SubjectRouting
        SubjectRouting --> Listening : 顺序命中听力
        SubjectRouting --> Reading : 顺序命中阅读
        SubjectRouting --> Writing : 顺序命中写作
        SubjectRouting --> Speaking : 顺序命中口语

        Listening --> SubjectRouting : 单科完成
        Reading --> SubjectRouting : 单科完成
        Writing --> SubjectRouting : 单科完成
        Speaking --> SubjectRouting : 单科完成

        state Listening {
            [*] --> LPart1
            LPart1 --> LPart2 : 进入下一Part
            LPart2 --> LPart3 : 进入下一Part
            LPart3 --> LPart4 : 进入下一Part
            LPart4 --> [*] : 听力完成
        }

        state Reading {
            [*] --> RPart1
            RPart1 --> RPart2 : 进入下一Part
            RPart2 --> RPart3 : 进入下一Part
            RPart3 --> [*] : 阅读完成
        }

        state Writing {
            [*] --> WTask1
            WTask1 --> WTask2 : 进入下一任务
            WTask2 --> [*] : 写作完成
        }

        state Speaking {
            [*] --> SPart1
            SPart1 --> SPart2 : 进入下一Part
            SPart2 --> SPart3 : 进入下一Part
            SPart3 --> [*] : 口语完成
        }
    }

    state Marking {
        [*] --> ObjectiveScoring
        ObjectiveScoring --> SubjectiveMarking : 听力/阅读自动判分完成
        SubjectiveMarking --> ScoreAggregation : 写作/口语人工或AI辅助阅卷完成
        ScoreAggregation --> [*] : 汇总四科成绩
    }

```
